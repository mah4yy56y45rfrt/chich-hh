# chich-hh
Цель проекта

Цель проекта - проверить, можно ли использовать PCA для сжатия аудиодорожки, а также оценить, как количество главных компонент влияет на качество восстановленного звука.

Как запустить проект

Открыть ноутбук в Jupyter Notebook или Google Colab.
Последовательно выполнить все ячейки.

Используемые технологии

- Python
- NumPy
- SciPy
- scikit-learn
- Matplotlib
- Jupyter Notebook / Google Colab

Результаты

PCA действительно можно использовать для сжатия аудио. При использовании 140 главных компонент из 1000 удалось сохранить около 99.69% объяснённой дисперсии, получить SNR около 25 dB и сжать данные примерно в 4.75 раза. На слух восстановленный сигнал остаётся похожим на исходный, хотя при сильном уменьшении числа компонент появляются шум и искажения.

Также стало понятно, что explained variance не всегда полностью отражает качество звука. Например, для сохранения 99% дисперсии достаточно 95 компонент, но на практике звук может требовать большего числа компонент, чтобы восприниматься качественно. Поэтому при работе с аудио важно оценивать результат не только по метрикам, но и на слух.
